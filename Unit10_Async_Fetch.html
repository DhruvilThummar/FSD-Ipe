Unit 10: Modern JavaScript - Async, Fetch & Forms

10.1 Form Validation & Storage

Form Validation:

  • HTML5 validation attributes: required, pattern, minlength, maxlength
  • Regular Expressions (Regex): /^[a-z]+$/, /[0-9]/, etc.
  • JavaScript validation: Check form before submission
  • Validate email, password strength, field lengths

Form Processing:

  • Get form values: document.getElementById('myInput').value
  • Submit event handler: form.addEventListener('submit', function(e) { })
  • Prevent default form submission: e.preventDefault()
  • Send data via AJAX or Fetch API

Web Storage:

  • localStorage: Persistent storage (survives page refresh)
  • sessionStorage: Temporary (cleared when tab closes)
  • Methods: setItem(key, value), getItem(key), removeItem(key), clear()
  • Store strings; use JSON.stringify() for objects

10.2 Asynchronous JavaScript & Promises

Promises: Handle async operations

  • Promise states: pending, fulfilled, rejected
  • .then(success, error) - Handle promise
  • .catch(error) - Catch rejection
  • .finally() - Execute regardless of result
  • Promise chaining: .then().then().catch()

Async/Await (ES7): Modern way to handle promises

  • async function - Declares async function
  • await - Pause execution until promise resolves
  • try...catch for error handling
  • Cleaner syntax than .then().catch()

10.3 Fetch API

Fetch API: Make HTTP requests

  • fetch(url) - Returns promise
  • .then(response => response.json()) - Parse JSON
  • Get requests: fetch(url)
  • Post requests: fetch(url, {method: 'POST', body: JSON.stringify(data)})
  • Error handling with .catch()
  • Common APIs: JSONPlaceholder, OpenWeather, REST APIs
