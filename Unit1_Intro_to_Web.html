Unit 1: Introduction to Web and Web Design

1.1 Basics of WWW and HTTP Protocol

Key Concepts:

  • WWW (World Wide Web) - A system of interconnected documents and resources
  • HTTP (HyperText Transfer Protocol) - Protocol for transferring data over the internet
  • HTTP Features: Stateless, Request-Response model, TCP/IP based
  • HTTP Request Methods: GET, POST, PUT, DELETE, PATCH
  • HTTP Status Codes: 1xx (Informational), 2xx (Success), 3xx (Redirection), 4xx (Client Error), 5xx (Server Error)
  • URL (Uniform Resource Locator) - Address of web resources
  • Client-Server Architecture: Client sends requests, Server responds

1.2 Concepts of Effective Web Design

Key Topics:

  • Website Planning: Define goals, target audience, content structure
  • Browser Compatibility: Design should work on different browsers
  • Bandwidth and Cache: Optimize for faster loading
  • Display Resolution: Responsive design for all screen sizes
  • Look and Feel: Color schemes, typography, visual hierarchy
  • Page Layout: Proper arrangement of content
  • Navigation Design: User-friendly menu and navigation structure

Practice Questions & Exercises

Q1: Explain the HTTP request-response cycle. What happens when you type a URL in your browser?
Q2: Differentiate between GET and POST methods in HTTP.
Q3: What do the following HTTP status codes mean? 200, 404, 500, 301
Q4: Explain Client-Server architecture with a real-world example.
Q5: How would you make a website compatible with different browsers? List at least 5 strategies.
Q6: Design a navigation structure for an e-commerce website. What elements would you include?
Q7: Explain how bandwidth affects website performance and what steps you can take to optimize it.
Q8: Create a sitemap for a blog website with multiple categories and pages.
